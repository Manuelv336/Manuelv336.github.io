Lab 02 Reflection: HTML

Learning HTML basics is fun and I think I've got the hang of it. Playing with the tags felt like building a pyramid: I started with the foundation html tag, moved up to the head and body, and then the rest of the levels, such as the header, main, and footer. The body is the main part. I will use the Google Fonts website in the list to see what fonts I’ll add to my future websites, and I use the playground as a sandbox to quickly see how a website looks when I'm testing my code. W3Schools is my go-to website to research different looks and options of html tags.

Things I learned:

  1. How to add basic HTML like h1, body, and header
  2. How to add and nest lists-ul/ol
  3. How to create and apply classes

Resources used:

  • Google Fonts
  • MDN Playground

Lab 03 Reflection: External CSS

A challenge I encountered: It was difficult to design from scratch without a template, but I knew I needed to add CSS to the body, headings, paragraphs, links and one class and ID selector. I found it a bit hard to customize the entire body because it can not be the same background color as the color in other fields. I also struggled with an ID selector to link.

How I worked through it: I used examples from the course readings and my personal notes. I am getting comfortable with basic CSS like colors, bold/italic text, and fonts, but I still rely on my notes for more technical things like borders, lines, and margins. I like it when things look like a box and soft color. The exercise said that we need to add one style to one element, but I played with multiple styles, and it worked for what I liked.

Resources used:

For the layout, I use some of the examples in the reading and my personal notes.

I read on MDN Web CSS RGBA on how to add background transparency.

MDN CSS RGBA

To understand and practice spacing, I use W3Schools, which talks about the CSS Box Model.

W3Schools CSS Box Model

Something that clicked: I realized how CSS is case sensitive. I also found that while basic styling, like background colors, is easy to remember, but the layout properties are harder to remember and require more practice.

Concept I'm still unsure about: I used box sizing for my .date-line, but I’m still confused about it. I understand that it has to do with how the width and height of an element are calculated, but I’m not sure when to use it and how it affects the layout of the page. I need to practice more with it to fully grasp its purpose and how it interacts with other CSS properties.

Things I learned:

  1. How to add colors and background styles
  2. Understanding the Box Model-padding, border and margin
  3. How to style inline, outline, and inside elements

Resources used:

  • MDN CSS RGBA
  • W3Schools CSS Box Model

Lab 05 Reflection: Flexbox

A challenge I encountered:

I struggled with the HTML structure at first, specifically leaving elements like my Things I Learned list outside of the flex boxes, which made the layout look broken. I also found it very difficult to decide on the right numbers for margins and padding. I ended up testing different numbers until I found something I liked, but I’m still struggling with getting the spacing exactly right.

How I worked through it:

To fix the layout, I looked at a practice example from Professor Roni’s website to see how the HTML tags were nested. I created a consistent box for each lab and added the resources and things I learned sections inside them to keep everything the same. I worked through the spacing issues by trial and error, adjusting the pixels until the content felt organized.

Something that clicked:

It clicked for me how Flexbox allows the content to expand and collapse dynamically. I really liked seeing how the containers respond to the screen size. I also realized that while I don't like how my design looks yet, I can see that it looks better than Lab 02. I’ve learned that I can eventually create templates to keep a consistent look based on what has worked well in the past. I also like how I can apply the hover effect to the main heading list.

A question or concept you are still unsure about:

I am still unsure if there is a standard list of numbers for padding and margins that we should not exceed or a lowest value we should stay above. I wonder if Web designers follow a specific guide for these measurements, or if it is always up to our own criteria and what we think looks best. Also, is the font size correct? I know for cell phone it looks good but not sure for large monitors.

Things I learned:

  1. Implementing Flexbox and Grid layouts
  2. make the website cell phone and computer visible
  3. Creating responsive designs with media queries
  4. Add responsive media queries
  5. Attach multiple css files

Resources used:

  • Guide to Flexbox
  • MDN-Responsive Design
  • Professor's Flexbox Examples
